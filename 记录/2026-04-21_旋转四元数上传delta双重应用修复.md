# 2026-04-21 旋转四元数上传时 delta 双重应用修复

## 提交信息
- **分支**: `master`
- **改动文件**: `src/App.vue`

---

## 一、问题现象

使用滑条旋转模型后点击"上传结果"保存数据，再导入该数据，发现模型的旋转角度与保存时完全不同。

---

## 二、根因分析

### Bug 1：slider delta 被双重应用（主因）

数据流时序：

1. 用户拖滑条 → `applySlidersToCamera()`（ThreeScene.vue:409）增量旋转 `modelGroup.quaternion`
2. 松开滑条 → `saveCurrentCameraRotationToProps()`（ThreeScene.vue:661）把**累积后的四元数**写入 `scene.cameraRotation`
3. 同时 localStorage 中的 slider delta（如 `pitchDelta: 15`）**没有清零**
4. 点击"上传结果" → `getCorrectedQuaternion()`（App.vue:641）读取 `scene.cameraRotation`（已是累积值）作为 base
5. 再从 localStorage 读取 delta，执行 `composeQuaternionFromBaseAndDeltas(累积值, 15, 0, 0)`
6. **delta 被叠加了两次**

### Bug 2：增量旋转 vs 一次性合成不等价（次因）

运行时 `applySlidersToCamera()`：
```js
// 每次拖动产生小增量 dp，绕模型当前局部轴旋转
modelGroup.quaternion.multiply(Rx(dp));  // 局部轴随每次旋转变化
```

上传时 `composeQuaternionFromBaseAndDeltas()`：
```js
// 把总 pitch/yaw/roll 作为一次性旋转合成
localDelta = Rx(totalPitch) * Ry(totalYaw) * Rz(totalRoll);
return base * localDelta;
```

多次小增量绕**不断变化的局部轴**旋转的累积效果 ≠ 一次性绕**固定轴**旋转总角度。旋转不可交换，两种方式产生不同结果。

---

## 三、修复内容

### `src/App.vue` — `getCorrectedQuaternion()`

`scene.cameraRotation` 已由 `ThreeScene.saveCurrentCameraRotationToProps()` 在每次滑条释放时写入正确的累积四元数。上传时直接使用即可，不再做 delta 合成。

```js
// 修复前（App.vue:641-691）：
function getCorrectedQuaternion(scene) {
  const rot = scene.cameraRotation;
  const qBase = new THREE.Quaternion(rot[0], rot[1], rot[2], rot[3]);
  // 从 localStorage 读 delta...
  // pitchDelta, yawDelta, rollDelta
  return composeQuaternionFromBaseAndDeltas(qBase, pitchDelta, yawDelta, rollDelta);
  // ↑ delta 被双重应用！qBase 已经包含了 delta 的效果
}

// 修复后：
function getCorrectedQuaternion(scene) {
  const rot = scene.cameraRotation;
  // 直接返回累积后的四元数，不再做 delta 合成
  return [
    Number(rot[0]) || 0,
    Number(rot[1]) || 0,
    Number(rot[2]) || 0,
    Number(rot[3]) || 1,
  ];
}
```
