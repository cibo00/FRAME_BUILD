# 2026-04-21 点云扫掠角度方向修复（AGH/JDI 填充曲面不过 D'）

## 提交信息
- **分支**: `master`
- **改动文件**: `src/components/ThreeScene.vue`

---

## 一、问题现象

点云扫掠生成的填充曲面无法覆盖到 D' 端点位置。AGH/JDI 侧轮廓插值扫掠产生的点云只覆盖了 A 端附近区域，D' 端完全缺失。

---

## 二、根因分析

前端 `ThreeScene.vue` 第 2285 行的扫掠角度公式与 Python 后端 `nailmodel.py` `_filling_surface()` 的角度方向相反。

### Python 后端（正确）

```python
# nailmodel.py lines 656-674
angle_margin = AD_arc / 10
angle_start = -angle_margin
angle_end   = AD_arc + angle_margin
angle = angle_start + tv * (angle_end - angle_start)
```

展开：`angle = -margin + tv * (AD_arc + 2*margin)`

| tv | angle |
|---|---|
| 0 | `-margin`（A 端稍前） |
| 1 | `+AD_arc + margin`（D' 端稍后） |

扫掠方向：从 A 端 → D' 端 ✓

### 前端（修复前，错误）

```js
// ThreeScene.vue line 2285（旧）
const angle = _angleMgn - tv * (Math.abs(AD_arc) + 2 * _angleMgn);
```

展开：`angle = +margin - tv * (|AD_arc| + 2*margin)`

| tv | angle |
|---|---|
| 0 | `+margin` |
| 1 | `-|AD_arc| - margin` |

扫掠方向：从 +margin → -(|AD_arc|+margin)，即从 A 端往 D' 的**反方向**走 ✗

当 `AD_arc > 0` 时，前端扫掠终点为 `-AD_arc - margin`，完全到不了 D' 所在的 `+AD_arc` 位置。

---

## 三、修复内容

### `src/components/ThreeScene.vue` line 2285

```js
// 修复前：
const angle = _angleMgn - tv * (Math.abs(AD_arc) + 2 * _angleMgn);

// 修复后：
const angle = -_angleMgn + tv * (Math.abs(AD_arc) + 2 * _angleMgn);
```

修复后展开：`angle = -margin + tv * (|AD_arc| + 2*margin)`

| tv | angle |
|---|---|
| 0 | `-margin`（A 端稍前） |
| 1 | `+|AD_arc| + margin`（D' 端稍后） |

与 Python `_filling_surface` 完全一致。

## 六、二次修复：扫掠旋转轴方向不一致（2026-04-21 追加）

### 问题

修复角度公式后，点云仍然不过 D'。

### 根因

前端有两个旋转函数，旋转方向约定不同：

| 函数 | 实际旋转角 | 使用场景 |
|---|---|---|
| `rotateVector`（1560 行） | `-theta`（内部取负） | 弧绘制（2126 行） |
| `rotatePointsAroundAxis`（1568 行） | `+theta`（不取负） | 扫掠（2290 行） |

`AD_arc_normal` 方向与 Python `arc_geometry.normal` 相反（前端 `AD × Z` = `-normal`）。

- 弧绘制：`rotateVector(OA, AD_arc_normal, theta)` → 实际绕 `AD_arc_normal` 旋转 `-theta` = 绕 `normal` 旋转 `+theta` ✓
- 扫掠：`rotatePointsAroundAxis(..., AD_arc_normal, angle)` → 实际绕 `AD_arc_normal` 旋转 `+angle` = 绕 `normal` 旋转 `-angle` ✗

弧绘制通过 `rotateVector` 的 `-theta` 自动补偿了方向差异，但扫掠用 `rotatePointsAroundAxis`（无 `-theta`），导致点云往 D' 的反方向旋转。

### 方向验证（归一化场景 A=(0,0,0), D=(1,0,0)）

```
Python:  normal = cross(OA, OD) / |...| = (0, +1, 0)
前端:    AD_arc_normal = AD × Z = (0, -1, 0) = -normal

绕 (0,-1,0) 旋转 +AD_arc  ≠  绕 (0,+1,0) 旋转 +AD_arc
绕 (0,-1,0) 旋转 -AD_arc  =  绕 (0,+1,0) 旋转 +AD_arc  ✓
```

### 修复

`ThreeScene.vue:2290`：

```js
// 修复前：
const rotated = rotatePointsAroundAxis(sweepPts, O, AD_arc_normal, angle);

// 修复后：
// rotatePointsAroundAxis 不含 -theta（与 rotateVector 不同），需取反以匹配弧方向
const rotated = rotatePointsAroundAxis(sweepPts, O, AD_arc_normal, -angle);
```
