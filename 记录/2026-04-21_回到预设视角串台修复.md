# 2026-04-21 "回到预设视角"切换数据组时串台修复

## 提交信息
- **分支**: `master`
- **改动文件**: `src/components/ThreeScene.vue`

---

## 一、问题现象

在管理区切换不同数据组后，点击"回到预设视角"按钮，模型旋转到的角度是**上一个数据组**的预设视角，而非当前数据组的预设视角。

---

## 二、根因分析

### 根因 1：`originalBackendRotationMap` 在切换数据组时从不清除

`originalBackendRotationMap`（`ThreeScene.vue:311`）是一个 `Map<string, number[]>`，key 是 `backgroundImage` URL。

- 当用户在管理区点击不同数据组时，`App.vue` 的 `loadDataByGroupId` 替换了 `scenesData.value`
- 但 ThreeScene 组件实例**不会被销毁重建**（SidePanel 中没有用 `:key` 绑定到 groupId），所以这个 Map 一直保留旧数据
- `watch(() => props.sceneData)` 中的写入逻辑有 guard：`if (!originalBackendRotationMap.has(newKey))`
- 如果两个不同数据组的图片 URL 相同，Map 里已有旧数据组的旋转值，新数据组的旋转值就**不会被写入**

### 根因 2：localStorage 也按 `backgroundImage` URL 做 key，同样不清除

`getOriginalRotationForScene` 从 `localStorage[ORIGINAL_ROTATION_KEY]` 读取，key 也是 `backgroundImage` URL。即使清了内存 Map，localStorage 里的旧值也会被当作"原始值"读回来。

### 根因 3：`getSceneKey()` 用 `backgroundImage` URL 做 key 不够唯一

不同数据组可能包含相同的图片 URL，但对应不同的旋转四元数。

---

## 三、修复方案

**方案 A：切换数据组时清空缓存**（已采用）

在 `watch(() => props.sceneData)` 回调中，每次 sceneData 变化时：
1. 清空 `originalBackendRotationMap`（`.clear()`）
2. 清空 localStorage 中的 `ORIGINAL_ROTATION_KEY`
3. 清空 localStorage 中的 `CAMERA_DELTA_MAP_KEY`
4. 用新数据的 `cameraRotation` 重新填充 Map 和 localStorage

同时简化 `applyCameraRotationFromArray` 中的 guard 逻辑，移除从 localStorage 读旧值的 fallback。

---

## 四、具体改动

### 改动 1：`watch(() => props.sceneData)` — 清空旧缓存并重新填充

**位置**: `ThreeScene.vue:1055-1066`

```typescript
// --- 修复"回到预设视角"串台问题 ---
// 每次 sceneData 变化时清空旧的原始旋转缓存（内存 + localStorage），
// 让新数据的旋转值重新写入，避免不同数据组之间因 key 相同而读到旧值。
originalBackendRotationMap.clear();
try { localStorage.removeItem(ORIGINAL_ROTATION_KEY); } catch (e) {}
try { localStorage.removeItem(CAMERA_DELTA_MAP_KEY); } catch (e) {}

// 后端数据到达时保存原始四元数（清空后一定会写入）
if (newKey && newVal?.cameraRotation && ...) {
  originalBackendRotationMap.set(newKey, newVal.cameraRotation.slice());
  setRotationMapEntry(newKey, newVal.cameraRotation.slice());
}
```

### 改动 2：`applyCameraRotationFromArray` — 简化 guard 逻辑

**位置**: `ThreeScene.vue:615-619`

移除了从 `getOriginalRotationForScene`（localStorage）读旧值的 fallback，简化为：

```typescript
const key = getSceneKey();
if (key && !originalBackendRotationMap.has(key)) {
  originalBackendRotationMap.set(key, newRotation.slice());
}
```

## 五、影响范围

- 同一数据组内切换图片时，Map 也会被清空，但会立即被新图片的旋转值重新填充，**无功能影响**
- 滑条 delta 在切换数据组后归零（`CAMERA_DELTA_MAP_KEY` 被清空），这是**预期行为**
