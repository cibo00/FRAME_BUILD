# 2026-04-21 D' 坐标系映射修复（前端未归一化导致 D' 位置偏移）

## 提交信息
- **分支**: `master`
- **改动文件**: `src/components/ThreeScene.vue`

---

## 一、问题现象

检查 D' 是否与 A 重合时发现：D' 与 A 不会重合（X 坐标不同），但 D' 的坐标计算存在坐标系映射 bug——前端直接照搬了 Python 归一化坐标系的公式，而前端使用的是未归一化的原始坐标。

---

## 二、根因分析

### 坐标系差异

| | Python（归一化后） | 前端（原始坐标） |
|---|---|---|
| A | `(0, 0, 0)` | `(A.x, A.y, 0)` — A.y 通常不为 0 |
| D | `(span, 0, 0)` | `(D.x, D.y, 0)` — D.y 通常不为 0 |
| AD 方向 | 沿 +X 轴 | XY 平面内任意方向 |
| 高度方向 | Z 轴 | Z 轴 |

Python 在 `_solve_ad_arc_from_tangent` 中做了归一化（A 移到原点、AD 对齐 +X），所以 `span_x = D.x - A.x` 就是 AD 的完整长度，D' 的 Y=0 也是正确的（AD 在 XZ 平面内）。

前端没有做归一化，导致两个错误：

### Bug 1：`span_x` 只取了 X 分量差

```js
// 修复前（ThreeScene.vue:2096-2097）
const a_x = A.x, a_z = A.z, d_x = D.x;
const span_x = d_x - a_x;  // 只有 X 分量差，丢失了 Y 分量
```

当 AD 方向不沿 X 轴时，`span_x` 小于 AD 的真实长度，导致 D' 的 Z 偏移量偏小。

### Bug 2：D' 的 Y 坐标硬编码为 0

```js
// 修复前（ThreeScene.vue:2102）
const dPrime = new THREE.Vector3(d_x, 0, d_prime_z);  // Y=0 错误
```

Python 中 Y=0 正确（归一化后 AD 在 XZ 平面），但前端 A.y 和 D.y 通常不为 0。D' 应该在 D 的 XY 位置上方（Z 方向偏移），而不是被强制拉到 Y=0 平面。

这导致 `chord = dPrime - A` 产生虚假的 Y 分量 `-A.y`，进而导致：
- `chordLen` 偏大
- `AD_arc_radius` 偏大
- 圆心 O 位置偏移
- 整个 AD 弧几何变形

### Bug 3：`perp` 向量硬编码了 XZ 平面旋转

```js
// 修复前（ThreeScene.vue:2111）
const perp = new THREE.Vector3(chord.z, 0, -chord.x);
```

这是在 XZ 平面内对 chord 做 90° 旋转（忽略 Y 分量），等价于 Python 归一化坐标系中的 `cross(normal, chord)`。但前端的 chord 是 3D 向量（有 X、Y、Z 分量），需要在弧平面内做旋转。

---

## 三、AD_arc_normal 方向验证

修复 `perp` 前需要确认前端 `AD_arc_normal` 与 Python `normal` 的方向关系。

### Python `normal`

归一化后 A、D'、center 都在 XZ 平面（Y=0），`normal = cross(OA, OD)` 结果沿 +Y 方向：

```
normal = (0, +1, 0)
```

### 前端 `AD_arc_normal`

```js
AD_arc_normal = AD × (0, 0, 1)
```

当 AD 沿 +X 时：`(1,0,0) × (0,0,1) = (0, -1, 0)`

结论：**前端 `AD_arc_normal` = -Python `normal`**（方向相反）。

### 对 `perp` 的影响

Python: `perp = cross(normal, chord)`

前端需要：`perp = cross(normal, chord) = cross(-AD_arc_normal, chord) = cross(chord, AD_arc_normal)`

所以前端的 `perp = chord × AD_arc_normal`。

## 四、修复内容

### 修改 1：D' 坐标计算（`ThreeScene.vue:2096-2105`）

```js
// 修复前：
const a_x = A.x, a_z = A.z, d_x = D.x;
const span_x = d_x - a_x;
const sinBeta = Math.sin(aTangent);
const d_prime_z = a_z + span_x * (Math.cos(aTangent) / sinBeta);
const dPrime = new THREE.Vector3(d_x, 0, d_prime_z);

// 修复后：
const adXY = new THREE.Vector2(D.x - A.x, D.y - A.y);
const span = adXY.length();
const sinBeta = Math.sin(aTangent);
const d_prime_z = A.z + span * (Math.cos(aTangent) / sinBeta);
const dPrime = new THREE.Vector3(D.x, D.y, d_prime_z);
```

### 修改 2：`perp` 向量（`ThreeScene.vue:2114-2115`）

```js
// 修复前：
const perp = new THREE.Vector3(chord.z, 0, -chord.x);

// 修复后：
const perp = new THREE.Vector3().crossVectors(chord, AD_arc_normal);
```

## 五、数学验证

### 验证 1：归一化场景（A=(0,0,0), D=(1,0,0)）

修复后：
- `span = |(1,0)| = 1` ✓（与 Python `span_x = 1` 一致）
- `dPrime = (1, 0, d_prime_z)` ✓（与 Python 一致）
- `chord = (1, 0, d_prime_z)`
- `AD_arc_normal = (0, -1, 0)`
- `perp = chord × AD_arc_normal = (1,0,dpz) × (0,-1,0) = (0*0-dpz*(-1), dpz*0-1*0, 1*(-1)-0*0) = (dpz, 0, -1)`

Python: `perp = (chord[2], 0, -chord[0]) = (dpz, 0, -1)` ✓ 完全一致。

### 验证 2：非归一化场景（A=(0,1,0), D=(1,2,0)）

修复前：
- `span_x = 1`，`dPrime = (1, 0, dpz)`
- `chord = (1, -1, dpz)` — Y 分量 -1 是虚假的

修复后：
- `span = |(1,1)| = √2`，`dPrime = (1, 2, dpz')`（dpz' 更大因为 span 更大）
- `chord = (1, 1, dpz')` — Y 分量正确反映 AD 方向
