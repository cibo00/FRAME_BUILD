## 功能说明

### 1. 自动心跳检测
- 用户登录成功后，自动启动心跳检测
- 每30秒向后端发送一次心跳请求
- 保持用户在线状态

### 2. 自动保存临时操作
- 监听共享点数据（positions、bezierPoints）和场景数据的变化
- 数据变化时自动保存到后端（带500ms防抖）
- 保存的数据包括：
  - positions: 点位置数据
  - bezierPoints: 贝塞尔点数据
  - scenes: 场景数据
  - globalADArc: 全局AD弧度
  - timestamp: 时间戳

### 3. 自动恢复状态
- 用户登录后，自动尝试从后端恢复临时操作
- 如果有保存的数据，自动恢复到之前的状态
- 恢复的数据包括点位置、场景等

### 4. 清除临时操作
- 用户点击"获取下一组"按钮时，自动清除临时操作
- 清除后获取新的数据组

### 5. 资源清理
- 页面关闭或组件卸载时，自动停止心跳检测
- 释放相关资源

## 工作流程

### 登录流程
```
用户登录
  → 保存session_token到localStorage
  → 初始化OperationManager
  → 启动心跳检测（每30秒）
  → 尝试恢复临时操作
  → 如果有保存的数据，恢复状态
```

### 操作保存流程
```
用户拖动点位置
  → 触发数据变化
  → watch监听器检测到变化
  → 调用operationManager.saveTempOperation()
  → 防抖500ms后发送保存请求到后端
```

### 刷新恢复流程
```
用户刷新页面
  → 页面加载
  → 检查localStorage中的session_token
  → 如果存在，初始化OperationManager
  → 调用restoreTempOperation()
  → 从后端获取保存的数据
  → 恢复点位置和场景状态
```

### 获取下一组流程
```
用户点击"获取下一组"
  → 调用fetchNextDataBatch(true)
  → 清除临时操作
  → 从后端获取新数据
  → 更新场景
```

## 技术细节

### OperationManager类

**主要方法：**
- `init()`: 初始化，启动心跳并恢复临时操作
- `startHeartbeat()`: 启动心跳检测
- `stopHeartbeat()`: 停止心跳检测
- `saveTempOperation(data)`: 保存临时操作（带防抖）
- `restoreTempOperation()`: 恢复临时操作
- `clearTempOperation()`: 清除临时操作
- `destroy()`: 销毁管理器，清理资源

**特性：**
- 自动防抖（500ms）避免频繁保存
- 完整的错误处理
- 详细的日志输出
- TypeScript类型支持

### 数据监听

使用Vue 3的`watch`监听器，监听以下数据变化：
- `sharedPointsState.positions`
- `sharedPointsState.bezierPoints`
- `scenesData.value`

配置：
- `deep: true`: 深度监听对象内部变化
- 自动防抖：避免频繁触发保存

## 测试建议

### 1. 心跳测试
1. 登录系统
2. 打开浏览器开发者工具 → Network标签
3. 观察是否每30秒发送一次`/api/heartbeat`请求
4. 检查请求是否成功（状态码200）

### 2. 保存测试
1. 登录系统
2. 拖动点位置
3. 在Network标签中观察是否发送`/api/temp-operation/save`请求
4. 检查请求是否成功

### 3. 恢复测试
1. 登录系统
2. 拖动点位置到某个位置
3. 等待保存完成（观察Network标签）
4. 刷新页面
5. 检查点位置是否恢复到之前的位置

### 4. 清除测试
1. 登录系统
2. 拖动点位置
3. 点击"获取下一组"按钮
4. 在Network标签中观察是否发送`/api/temp-operation/clear`请求
5. 检查是否获取了新数据

### 5. 超时测试
1. 登录系统
2. 拖动点位置
3. 停止所有操作，等待超过1分钟
4. 刷新页面
5. 检查是否没有恢复之前的状态（因为后端已清理）

## 注意事项

1. **Session Token**: 确保用户登录后才能使用心跳和临时操作功能
2. **网络状态**: 如果网络断开，心跳和保存请求会失败，但不会影响正常使用
3. **数据大小**: 注意保存的数据大小，避免过大的JSON数据
4. **浏览器兼容性**: 使用了现代浏览器API，建议使用Chrome、Firefox、Edge等现代浏览器
5. **防抖时间**: 保存操作有500ms防抖，频繁操作时会合并保存请求

## 故障排查

### 问题1: 心跳请求失败
- 检查后端服务是否启动
- 检查session_token是否有效
- 检查网络连接

### 问题2: 保存失败
- 检查数据格式是否正确
- 检查后端API是否正常
- 查看浏览器控制台错误信息

### 问题3: 刷新后无法恢复
- 检查是否在1分钟内有心跳
- 检查是否点击了"获取下一组"（会清除临时操作）
- 检查session_token是否一致

### 问题4: 编译错误
- 确保安装了所有依赖：`npm install`
- 检查TypeScript版本
- 查看编译错误信息
