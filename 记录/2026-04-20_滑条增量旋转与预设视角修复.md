## 二、核心改动

### 1. 滑条增量旋转改为绕模型当前自身轴（增量 delta 模式）

**问题**：之前 `composeQuaternionFromBaseAndDeltas` 每次从 base 重新计算 `Q_base * Q_pitch * Q_yaw * Q_roll`，所有增量绕 base 的固定轴旋转。调整 roll 后再调 pitch，pitch 仍绕原始 base X 轴而非模型当前 X 轴。

**修复**：删除 `composeQuaternionFromBaseAndDeltas`，改为增量 delta 模式：

```ts
// 跟踪上次滑条值
let prevPitchVal = 0, prevYawVal = 0, prevRollVal = 0;

function applySlidersToCamera() {
  const dp = pitchValue.value - prevPitchVal;
  const dy = yawValue.value - prevYawVal;
  const dr = rollValue.value - prevRollVal;

  // 每个非零 delta 绕模型当前局部轴旋转（右乘）
  if (dp !== 0)
    modelGroup.quaternion.multiply(
      new THREE.Quaternion().setFromAxisAngle(new THREE.Vector3(1,0,0), degToRad(dp)));
  // yaw / roll 同理...

  prevPitchVal = pitchValue.value;
  // ...
}
```

关键：`modelGroup.quaternion.multiply(deltaQ)` 是右乘，deltaQ 的轴在模型**当前**局部空间中解释。所以每次拖动滑条都是绕模型**当前实时**的 X/Y/Z 轴旋转。

### 2. 后端原始四元数保护机制

**问题**：`originalBackendRotationMap` 和 localStorage 中的原始旋转值被累积四元数覆盖。

**污染链路**：
1. `saveCurrentCameraRotationToProps` 把累积四元数写入 `props.sceneData.cameraRotation`
2. `saveStateToLocalStorage()` 持久化累积值到 localStorage
3. 页面刷新后 App.vue 从 localStorage 恢复 → `scenesData` 包含累积值
4. `getOriginalRotationForScene` fallback 到 `props` → 读取到累积值
5. 累积值被当作"原始值"存入 `originalBackendRotationMap` 和 localStorage

**修复**：

1. 新增 `originalBackendRotationMap`（内存 Map），存储后端原始四元数，不被滑条操作覆盖
2. `getOriginalRotationForScene` 不再 fallback 到 props（props 可能已被累积值覆盖）
3. `applyCameraRotationFromArray` 仅在首次调用时存储原始值，且优先从 localStorage 读取
4. `setRotationMapEntry` 仅在 localStorage 无记录时写入
5. 场景 watcher 中后端数据首次到达时立即保存原始四元数
6. 每次挂载清除已污染的原始旋转缓存（`localStorage.removeItem(ORIGINAL_ROTATION_KEY)`）

### 3. "回到预设视角"按钮修复

**问题**：`resetCameraToProps` 从 `props.sceneData.cameraRotation` 读取，但该值已被滑条累积值覆盖。

**修复**：
- 优先从 `originalBackendRotationMap` 读取后端原始四元数
- 重置后同步更新 `props.sceneData.cameraRotation` 为原始值
- 调用 `resetSliderTracking()` 将增量跟踪变量重置为 0

### 4. 滑条松手后不重置

**问题**：之前 `saveCurrentCameraRotationToProps`（`@change` 触发）会将所有三个滑条重置为 0。

**修复**：移除滑条重置代码，保持各轴增量值。滑条值仅通过"回到预设视角"按钮清零。

### 5. 场景切换后姿态正确恢复

**问题**：重置后切场景再切回来，模型仍显示滑条调整后的姿态。

**根因**：`resetCameraToProps` 没有把原始四元数写回 `props`，切换场景时 watcher 从 props 读到累积值。且 watcher 仅在 deltas 非 0 时调用 `applySlidersToCamera()`。

**修复**：
- `resetCameraToProps` 同步更新 `props.sceneData.cameraRotation` 为原始值
- 两处 watcher 都改为始终调用 `resetSliderTracking()`（确保 prev 值正确）

### 6. 右键禁用

```ts
controls.enablePan = false;  // 禁用右键平移
// 已有：
controls.enableZoom = false;
controls.enableRotate = false;
```

### 7. D 点在尖侧轮廓中位置修复

**问题**：`syncSideProfilePointsFromStore()` 将 D 的坐标从 `sharedPointsState.positions`（主模型坐标 x=16.045）直接赋给尖侧轮廓，但尖侧轮廓使用的是局部坐标系（x=宽度方向，范围约-6.5~+7），D 应为原点 (0,0,0)。

**修复**：将 D 点来源改为固定值 `{ x: 0, y: 0, z: 0 }`。

### 8. 点云左右半边控制映射修正

**问题**：视角从 D→A 改为 A→D 后，`_perpAD`（宽度方向向量）的左右方向需要取反。

**修复**：
```ts
const _perpAD = _zHat.clone().cross(AD.clone().normalize()).negate(); // 视角从A看D，左右取反
```

### 9. D 点 (0,0,0) 防止泄漏到主模型

**问题**：`handleTipPointUpdate` 中 `pointName === 'D'` 分支将 `newPosition` 写入 `sharedPointsState.positions`，但尖侧轮廓中 D 的位置是 (0,0,0)。

**修复**：添加条件判断，仅当 `newPosition` 非零时才写回主模型坐标：
```ts
if (sharedPositionPoint && (newPosition.x !== 0 || newPosition.y !== 0 || newPosition.z !== 0)) {
  // 写回
}
```

## 三、涉及的关键数据结构

### localStorage keys

| Key | 内容 | 说明 |
|---|---|---|
| `STORAGE_KEY` | 场景元数据（含 `cameraRotation` 累积值） | 每次滑条松手时更新 |
| `ORIGINAL_ROTATION_KEY` | 每场景后端原始四元数 | 仅首次写入，挂载时清除 |
| `CAMERA_DELTA_MAP_KEY` | 每场景滑条增量 { pitch, yaw, roll } | 滑条拖动时实时更新 |

### 内存数据

| 变量 | 内容 |
|---|---|
| `originalBackendRotationMap` | Map<sceneKey, [x,y,z,w]> 后端原始四元数 |
| `prevPitchVal / prevYawVal / prevRollVal` | 上次滑条值（用于计算增量 delta） |

## 五、遗留说明

1. 增量 delta 模式下，滑条值与四元数之间不再有直接的分解关系。页面刷新后恢复依赖 localStorage 中保存的累积四元数 + 滑条值，首次加载时通过 `applyCameraRotationFromArray` + `resetSliderTracking` 初始化。
2. `App.vue` 中 `getCorrectedQuaternion()` 仍使用旧的 `composeQuaternionFromBaseAndDeltas` 公式计算导出四元数，这与前端显示用的增量 delta 模式不同。后续需统一。
3. 每次挂载时清除 `ORIGINAL_ROTATION_KEY` 是一次性的清理措施。后续版本稳定后可移除。
