## 功能一：对称点自动移动

### 文件：`src/components/ThreeScene.vue`
### 修改位置：第 2069-2143 行（dragend 事件处理器）

### 功能描述
在三维空间中移动某个点时，其对应的对称点会相对于中心点自动保持对称位置。

### 对称关系定义
| 点1 | 中心点 | 点2 | 说明 |
|-----|-------|-----|------|
| AB_P1 | A | AF_P1 | AB_P1 移动时 AF_P1 自动对称移动，反之亦然 |
| AB_P4 | B | BC_P1 | AB_P4 移动时 BC_P1 自动对称移动，反之亦然 |
| CD_P2 | D | ED_P2 | CD_P2 移动时 ED_P2 自动对称移动，反之亦然 |
| AF_P4 | F | FE_P1 | AF_P4 移动时 FE_P1 自动对称移动，反之亦然 |

### 对称计算公式
```
对称点坐标 = 2 × 中心点坐标 - 移动点坐标
```

对于每个轴（x, y, z）都应用此公式：
- `newSymmetricX = 2 * centerPoint.x - movedPoint.x`
- `newSymmetricY = 2 * centerPoint.y - movedPoint.y`
- `newSymmetricZ = 2 * centerPoint.z - movedPoint.z`

#### 修改后（新增对称点逻辑）
```typescript
const bezierIndex = bezier_points.findIndex(p => p === draggedObject);
if (bezierIndex !== -1) {
  // 如果是贝塞尔点，更新 bezierPoints 数据
  const pointData =  sharedPointsState.bezierPoints[bezierIndex];
  pointData.x = draggedObject.position.x;
  pointData.y = draggedObject.position.y;
  pointData.z = draggedObject.position.z;

  // ⭐ 新增：对称点移动逻辑
  // 定义对称关系：[点1名称, 中心点名称, 点2名称]
  const symmetricPairs = [
    ['AB_P1', 'A', 'AF_P1'],
    ['AB_P4', 'B', 'BC_P1'],
    ['CD_P2', 'D', 'ED_P2'],
    ['AF_P4', 'F', 'FE_P1']
  ];

  // 检查被拖拽的点是否在对称关系中
  for (const [point1Name, centerName, point2Name] of symmetricPairs) {
    let movedPointName = pointData.name;
    let symmetricPointName: string | null = null;
    let shouldUpdate = false;

    // 判断移动的点是哪一个，确定需要更新的对称点
    if (movedPointName === point1Name) {
      symmetricPointName = point2Name;
      shouldUpdate = true;
    } else if (movedPointName === point2Name) {
      symmetricPointName = point1Name;
      shouldUpdate = true;
    }

    if (shouldUpdate && symmetricPointName) {
      // 找到中心点的坐标
      const centerPoint = sharedPointsState.positions.find(p => p.name === centerName);
      if (centerPoint) {
        // 计算对称点的新位置：对称点 = 2 * 中心点 - 移动的点
        const newSymmetricX = 2 * centerPoint.x - pointData.x;
        const newSymmetricY = 2 * centerPoint.y - pointData.y;
        const newSymmetricZ = 2 * centerPoint.z - pointData.z;

        // 更新对称点的数据
        const symmetricPointData = sharedPointsState.bezierPoints.find(p => p.name === symmetricPointName);
        if (symmetricPointData) {
          symmetricPointData.x = newSymmetricX;
          symmetricPointData.y = newSymmetricY;
          symmetricPointData.z = newSymmetricZ;

          // 同时更新3D场景中对称点的位置
          const symmetricBezierIndex = sharedPointsState.bezierPoints.findIndex(p => p.name === symmetricPointName);
          if (symmetricBezierIndex !== -1 && bezier_points[symmetricBezierIndex]) {
            bezier_points[symmetricBezierIndex].position.set(newSymmetricX, newSymmetricY, newSymmetricZ);

            // 更新对称点的标签位置
            if (bezierLabels[symmetricBezierIndex]) {
              bezierLabels[symmetricBezierIndex].position.copy(computeLabelPosition(bezier_points[symmetricBezierIndex].position));
              const bLine = bezierLabelLines[symmetricBezierIndex];
              if (bLine) {
                (bLine.geometry as THREE.BufferGeometry).setFromPoints([
                  bezier_points[symmetricBezierIndex].position.clone(),
                  bezierLabels[symmetricBezierIndex].position.clone()
                ]);
                bLine.geometry.attributes.position.needsUpdate = true;
              }
            }

            // ⭐ 重新生成贝塞尔曲线以更新对称点对应的曲线
            generateCustomBezierCurves(false);
          }
        }
      }
      break; // 找到匹配的对称关系后退出循环
    }
  }

  // 持久化状态
  saveStateToLocalStorage();
}
```

### 关键实现要点
1. **对称点检测**：遍历 `symmetricPairs` 数组，检查被拖拽的点是否在对称关系中
2. **中心点查找**：从 `sharedPointsState.positions` 中查找对称中心点的坐标
3. **对称坐标计算**：使用对称公式计算对称点的新坐标
4. **数据更新**：
   - 更新 `sharedPointsState.bezierPoints` 中的对称点数据
   - 更新 3D 场景中对称点的实际位置
   - 更新对称点的标签位置和连线
5. **曲线重绘**：调用 `generateCustomBezierCurves(false)` 重新生成贝塞尔曲线
6. **状态持久化**：保存更新后的状态到 localStorage

### 工作流程
1. 用户拖拽点（例如 AB_P1）
2. `drag` 事件实时更新 AB 曲线
3. `dragend` 事件触发：
   - 更新被拖拽点的数据
   - 检测是否有对称关系
   - 如有，计算并更新对称点（AF_P1）的位置
   - 更新对称点的标签和连线
   - 重新生成所有贝塞尔曲线（包括 AF 曲线）
   - 保存状态

```typescript
function saveAllPointsData() {
    isSaving.value = true;

    try {
        // 定义点名称替换映射
        const nameReplaceMap: { [key: string]: string } = {
            'AB_P4': 'B_P_AB',
            'CD_P2': 'D_P_CD',
            'AF_P1': 'A_P_AF',
            'AF_P4': 'F_P_AF'
        };

        // 辅助函数：替换点名称
        const replacePointNames = (points: any[]) => {
            return points.map(point => {
                const newName = nameReplaceMap[point.name] || point.name;
                return { ...point, name: newName };
            });
        };

        // 1. 组装请求体 (Payload)，并替换点名称
        const payload: LocalSavePayload = {
            // 获取最新的共享点数据并替换名称
            positions: replacePointNames(sharedPointsState.positions),
            bezierPoints: replacePointNames(sharedPointsState.bezierPoints),
            // ... 其他场景数据
        };

        // 2-5. JSON序列化和下载（同原代码）
        // ...
    } catch (error) {
        // 错误处理
    } finally {
        isSaving.value = false;
    }
}
```

### 关键实现要点
1. **映射表定义**：创建 `nameReplaceMap` 对象存储原名称到新名称的映射关系
2. **辅助函数**：`replacePointNames` 函数遍历点数组，根据映射表替换名称
3. **数据处理**：
   - 对 `positions` 数组应用名称替换
   - 对 `bezierPoints` 数组应用名称替换
   - 不在映射表中的点保持原名称不变
4. **非侵入性**：仅影响导出的 JSON 文件内容，不修改内存中的实际数据
5. **保持引用安全**：使用扩展运算符 `{ ...point, name: newName }` 创建新对象，避免修改原对象

## 注意事项

1. **对称点移动仅在对称关系定义的点之间生效**，其他点不受影响
2. **曲线重绘**调用 `generateCustomBezierCurves(false)` 会重新生成所有曲线，确保视觉同步
3. **名称替换仅影响导出文件**，不会修改运行时的数据结构
4. **状态持久化**：对称点移动后会自动保存到 localStorage
5. **性能考虑**：曲线重绘在每次对称点移动后触发，如有性能问题可考虑节流优化
