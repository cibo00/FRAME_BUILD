# 开发记录 2026-04-16

## 需求来源

参照 `需求260416.md`：网页旋转摄像机视角后模型与剪影图片不匹配。

## 问题分析

### 数据流

1. `jointstest.py` 通过 WiLoR 检测手部姿态，计算手指局部坐标系的旋转矩阵并转为四元数 `[qx, qy, qz, qw]`
2. 四元数写入 `testdata.json` 的 `image_metadata_list`，与各剪影图片一一对应
3. `App.vue` 读取四元数，作为 `cameraRotation` 传递给 `ThreeScene.vue`
4. `ThreeScene.vue` 的 `applyCameraRotationFromArray` 将四元数转为相机旋转

### 根因定位

`ThreeScene.vue:441-519` 的 `applyCameraRotationFromArray` 函数在四元数模式下存在 **3 个数学错误**：

#### 错误 1：欧拉角分解后独立修改（最严重）

原代码流程：
```
四元数 → Euler(XYZ) → 取反X → 取反Y → 90°-Z → 新Euler(XYZ) → 新四元数
```

**问题**：欧拉角的三个分量是耦合的，不能独立取反/偏移来表示坐标变换。这仅在单轴旋转时碰巧正确，对复合旋转（本项目的大多数四元数）会产生严重偏差。

#### 错误 2：缺少取逆操作

Python 端四元数代表 **模型旋转**（旋转模型、固定相机来生成剪影）。Vue 端改为旋转相机、固定模型，数学上需要对旋转取逆（inverse）才能得到等效视角。原代码未做取逆。

#### 错误 3：坐标系转换不正确

Python `build_rotation_matrix_from_axes` 使用自定义坐标系：
- Target X = OpenCV Y（向下）
- Target Y = OpenCV X（向右）
- Target Z = -OpenCV Z（向观察者）

Three.js 坐标系：
- X = 向右，Y = 向上，Z = 向观察者

映射关系：Target → ThreeJS 等价于 **R_z(-90°)** 旋转。正确的转换需要对四元数做共轭变换（similarity transform），而非修改欧拉角。

## 修复方案

### 核心修改

文件：`src/components/ThreeScene.vue`，函数 `applyCameraRotationFromArray`

将四元数路径改为纯四元数运算（不经过欧拉角中间步骤）：

```typescript
// Step 1: 坐标系转换 — 共轭变换
// qBasis = R_z(-90°) 的四元数 = (√2/2, 0, 0, -√2/2)
const qBasis = new THREE.Quaternion(Math.SQRT1_2, 0, 0, -Math.SQRT1_2);
const qBasisInv = qBasis.clone().invert();
const qThreejsModel = qBasis.clone().multiply(qBackend).multiply(qBasisInv);

// Step 2: 取逆 — 模型旋转 → 相机旋转
targetQuaternion = qThreejsModel.clone().invert();
```

### 欧拉角模式

3 元素欧拉角路径保持不变，仅影响 4 元素四元数路径。

### `baseCameraEulerDeg` 兼容性

四元数模式下从最终相机四元数分解欧拉角写入 `baseCameraEulerDeg`，确保滑条（pitch/yaw/roll）增量叠加逻辑正常工作。

## 数学推导

### 坐标系映射矩阵

Python Target 坐标系在 Three.js 空间中的基向量：
- Target X = (0, -1, 0)（向下）
- Target Y = (1, 0, 0)（向右）
- Target Z = (0, 0, 1)（向观察者）

基变换矩阵：
```
P = [[0, 1, 0],    // new_x = old_y (Target.Y → ThreeJS.X)
     [-1, 0, 0],   // new_y = -old_x (Target.X → -ThreeJS.Y)
     [0, 0, 1]]    // new_z = old_z (Target.Z → ThreeJS.Z)
```

P = R_z(-90°)，对应四元数 q_basis = (cos(-45°), 0, 0, sin(-45°)) = (√2/2, 0, 0, -√2/2)

### 完整变换公式

```
Q_camera = (q_basis × Q_python × q_basis⁻¹)⁻¹
```

其中 × 为四元数乘法（Hamilton 积），⁻¹ 为四元数取逆（对于单位四元数等于共轭）。
